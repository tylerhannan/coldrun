# Q23

```sql
SELECT SearchPhrase, MIN(URL), MIN(Title), COUNT(*) AS c, COUNT(DISTINCT UserID) FROM hits WHERE Title LIKE '%Google%' AND URL NOT LIKE '%.google.%' AND SearchPhrase <> '' GROUP BY SearchPhrase ORDER BY c DESC LIMIT 10;
```

## Fast path

`group_fused_q23.rs` — **two-phase sort** (commit `18d7641`):

1. Parallel collect matching `(phrase_hash, UserID)` pairs; sort; run-length **count** + **distinct users** per phrase; heap top-10 by count.
2. Rescan matching rows for top-10 phrase hashes only — track `MIN(URL)`, `MIN(Title)`, phrase text.

Inline filters: `memchr` for `Google` in Title, reject `.google.` in URL.

## Why not hash maps

Pre-`18d7641` parallel path used `AHashSet<i64>` per SearchPhrase group. On 100M rows this OOM-killed `coldrun serve` (~18 GiB RSS) at Q23 and stopped the warm bench.
