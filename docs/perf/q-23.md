# Q23

```sql
SELECT SearchPhrase, MIN(URL), MIN(Title), COUNT(*) AS c, COUNT(DISTINCT UserID) FROM hits WHERE Title LIKE '%Google%' AND URL NOT LIKE '%.google.%' AND SearchPhrase <> '' GROUP BY SearchPhrase ORDER BY c DESC LIMIT 10;
```

## Fast path

`group_fused_q23.rs` — **disk streaming** (never loads URL+Title+SearchPhrase into warm-serve cache):

1. **Mask:** Title (`%Google%`) + URL (not `%.google.%`) — two column scans from disk.
2. **Count:** SearchPhrase scan — empty-filter + sharded `AHashMap<phrase_hash, count>` in one pass.
3. **Top-10** phrase hashes by count (heap).
4. **Detail:** one batched pass2 — SearchPhrase row lists for all top-10, then one URL scan + one Title scan + UserID for `COUNT(DISTINCT)`.

Seven column decompresses total (was ~35 with per-phrase reloads).

Warm `serve` **evicts** Q23 columns before execution (`drop_columns`) so Q23 is not stacked on ~33 GiB from Q1–22.
