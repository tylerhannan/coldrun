# Q36

```sql
SELECT ClientIP, ClientIP - 1, ClientIP - 2, ClientIP - 3, COUNT(*) AS c FROM hits GROUP BY ClientIP, ClientIP - 1, ClientIP - 2, ClientIP - 3 ORDER BY c DESC LIMIT 10;
```

## Fast path

`group_fused_q36.rs` → `group_columnar::clientip_quad_topk`.

Current path is adaptive:

- Sample `ClientIP` uniqueness.
- Use parallel hash counting + exact top-k tie ordering when duplicates are high enough.
- Fall back to sort/run-length when data looks near-unique.
- Unpack quad for SELECT list.

No filter (`WHERE` none) — scans full 100M ClientIP column.
