# Q24

```sql
SELECT * FROM hits WHERE URL LIKE '%google%' ORDER BY EventTime LIMIT 10;
```

## Adversarial

`SELECT *` materializes all columns for 10 rows — worst case for a columnar engine if implemented naïvely.

## Fast path

`scan_stream.rs` (dispatched before `open_table_for_query` when `q24_narrow_load`):

1. **Disk stream** URL column — LIKE filter + EventTime top-10 **heap** (no vector of all matching row indices).
2. **Evict** warm cache (`retain_columns(empty)`).
3. **`project_rows`** — sequential one-column LZ4 decode for 10 rows × ~80 columns.

`scan_fast.rs` fallbacks use the same top-K heap pattern for in-cache paths (demo).
